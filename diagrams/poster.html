How AI is Implemented & Technical Architecture
🧠 How AI is Implemented — LLM RCA Pipeline
📑
Model: gemma2:2b (Google Gemma, CPU-Optimised) 1.6 GB model loaded into Ollama. Runs CPU-only (no GPU required). Pulled once from HuggingFace, stored in PVC, survives pod restarts. Mirrors GDC air-gapped model serving.
LLM
🔒
Deployment: ClusterIP Only — Zero Egress Ollama Service on port 11434 as ClusterIP. NetworkPolicy restricts ingress to airflow namespace only. No external internet traffic. Identical to enterprise on-prem/GDC air-gapped pattern.
Air-Gapped
📄
Prompt Engineering: Structured JSON Forcing Prompt template forces format:"json" + strict schema. Temperature 0.1 for determinism. Includes error context, log excerpt (2000 chars), and 30-day historical baseline to improve accuracy.
Prompt
📈
Anomaly Detection: Statistical Baseline + LLM Explanation PostgreSQL view computes mean ± 2σ per DAG over 30 days. If a run drifts >20% from baseline, Ollama generates a natural-language explanation of likely cause before it becomes an SLA breach.

🔒
100% Air-Gapped — Zero Data Leaves the Cluster

All LLM inference runs on internal ClusterIP (Ollama :11434). No calls to OpenAI, Vertex AI, or any external service. Sensitive pipeline logs and business data never leave the Kubernetes cluster. Fully compliant with enterprise air-gap requirements and mirrors the GDC on-prem deployment pattern exactly.

☁
Lift-and-Shift Ready for Enterprise GDC

This PoC is architecturally identical to the production GDC deployment: Helm-managed Airflow, KubernetesExecutor, ephemeral Spark via Operator, ClusterIP LLM. Migrating to enterprise means changing a few endpoint URLs and scaling node pools — the DAG code, callbacks, and notification logic are identical.

📈
Proactive vs Reactive — Anomaly Detection

Beyond reactive RCA, the system continuously monitors DAG execution baselines (mean ± 2σ, 30-day rolling). When a DAG drifts more than 20% from its normal duration, Ollama generates a natural-language explanation and sends a proactive alert — before the SLA is breached.
